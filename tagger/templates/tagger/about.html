taglish part-of-speech tagger

This website contains tools for working with a part-of-speech tagger for the Taglish code-switched language. Taglish is considered a low-resource language in the field of Natural Language Processing, yet data is present everywhere in Filipino interactions, especially in popular social media platforms such as Facebook and Twitter.

The sole motivation behind this project is to propagate the language further in Natural Language endeavors and provide resources necessary to understanding and working with Taglish.

Builds on the Universal Dependencies (UD) Project

The Universal Dependencies (UD) Project provides a framework for consistent annotation of grammar (including parts-of-speech, of course) regardless of what language you are working with. You can go visit the project's website through this link.

The project makes use of the Universal POS Tags. This decision is based on the fact that this tagset works on both English and Tagalog, therefore it would be easy to apply in a code-switching setting between the two languages. You can visit this link.

How to use the online model

This web application contains a trained model which accepts a sentence and gives its predictions on its token's parts-of-speech.

  1. Input your sentence in the form.
  2. Click on the "Predict Tags" button.
  3. View the results (token are "badges" with colors). There is also a summary beside the annotation.
  4. (Optional) If you are logged in, you can add it to the website dataset by clickin on the "Add to Dataset" button.

Retraining facilities are provided for this online model, which can be accessed by account holders.

Website facilities (dataset, retraining)

The web application has facilities for storing the whole taglish dataset (and thus adding more sentences through an annotation tool).

You must be authenticated as a contributor to access the "Annotator" page, which holds these facilities.

Browsing the dataset

The dataset can be accessed with a table that is paginated. You can also search the dataset by identifier (ID column).

View a sentence's annotation by clicking on its eye button. A popup modal will show up.

Annotating a sentence

  1. Either edit a sentence from the dataset or fill-up the sentence information form, then click the "Start Annotating" button.
  2. Annotate each individual token with the tools provided.
  3. (OPTIONAL) To add a token, go to the "Add a Token" panel, type in your new token, then click on either of the insert buttons.
  4. (OPTIONAL) To edit the current token, go to the "Edit Current Token" panel then change accordingly. You can even delete this token. NOTE that you cannot delete a token if it's the last one in the sentence being annotated.
  5. Click on the "Save Annotation" button to submit it to the dataset.
